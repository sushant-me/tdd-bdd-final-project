# User Story Template

This document outlines the standard template used for defining user stories in this project. Each story is designed to capture the requirement from the perspective of the end user.

---

## **The Format**

**As a** [Role/Persona]  
**I want** [Action/Feature]  
**So that** [Benefit/Value]

---

## **Example User Story**

**As a** Product Manager  
**I want** to be able to list all products in the service  
**So that** I can see the current inventory and pricing at a glance.

---

## **Acceptance Criteria**

Each user story must meet the following criteria to be considered "Done":
1. The code passes all unit tests (TDD).
2. The feature is verified via BDD acceptance tests (Gherkin).
3. The code is linted and meets PEP8 standards.
4. The documentation/README is updated if necessary.
